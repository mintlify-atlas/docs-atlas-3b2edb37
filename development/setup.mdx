This guide walks you through everything you need to get Pi Day running on your machine — from cloning the repository to opening the interactive experience in your browser. Pi Day is a frontend-only React app built with Vite, so there is no backend or database to configure.

<Note>
  **Prerequisites** — before you begin, make sure you have the following installed:
  - **Node.js 18 or later** — [nodejs.org](https://nodejs.org)
  - **A package manager** — npm (bundled with Node.js), yarn, or pnpm
  - **Git** — [git-scm.com](https://git-scm.com)
</Note>

## Setup

<Steps>
  <Step title="Clone the repository">
    Run the following command to clone the repo and move into the project directory:

    ```bash
    git clone https://github.com/Fercho12s/pi-day-vercel.git && cd pi-day-vercel
    ```
  </Step>

  <Step title="Install dependencies">
    Install all required packages using your preferred package manager:

    <CodeGroup>
      ```bash npm
      npm install
      ```

      ```bash yarn
      yarn install
      ```

      ```bash pnpm
      pnpm install
      ```
    </CodeGroup>
  </Step>

  <Step title="Start the dev server">
    Start the Vite development server with hot module replacement (HMR):

    <CodeGroup>
      ```bash npm
      npm run dev
      ```

      ```bash yarn
      yarn dev
      ```

      ```bash pnpm
      pnpm dev
      ```
    </CodeGroup>

    By default, Vite serves the app at **http://localhost:5173**.
  </Step>

  <Step title="Open the app in your browser">
    Navigate to [http://localhost:5173](http://localhost:5173) to see the Pi Day interactive experience running locally. Changes you make to source files are reflected instantly via HMR — no manual refresh needed.
  </Step>
</Steps>

## Available scripts

The `package.json` defines three npm scripts for the full development lifecycle:

| Script | Command | Description |
|--------|---------|-------------|
| `dev` | `vite` | Starts the Vite dev server with HMR on port 5173 |
| `build` | `vite build` | Runs the TypeScript compile step and produces a production build in `dist/` |
| `preview` | `vite preview` | Serves the contents of `dist/` locally so you can verify the production build before deploying |

Run any script with your package manager:

```bash
npm run build
npm run preview
```

## TypeScript configuration

The project uses TypeScript with a `tsconfig.json` targeting ES2020. A path alias is configured so you can import from `src/` using the `@/` shorthand anywhere in the codebase:

```typescript
// Instead of this:
import { PiDisplay } from "../../components/PiDisplay";

// You can write this:
import { PiDisplay } from "@/components/PiDisplay";
```

The alias is defined in both `tsconfig.json` and `vite.config.ts`, so it works during type-checking and at bundle time.
