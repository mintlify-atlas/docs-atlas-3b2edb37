title: "Pi digit memory game: test your recall"
description: "A keyboard-driven game where you recite the digits of π one by one, choosing between three modes that trade lives, time pressure, and difficulty."
---

The **Memory Game** challenges you to recall the decimal digits of π in sequence. The app displays the digits you have already entered, highlights the current position with a blinking cursor, and waits for you to press the correct number key on your keyboard. Each correct digit earns points and advances your position; a wrong key costs a life (or ends the game outright in Endless mode). The goal is to see how far into π you can get before your lives or time run out.

## How to play

<Steps>
  <Step title="Choose a game mode">
    On the start screen you are presented with three mode cards. Pick the one that matches how much pressure you want.
  </Step>
  <Step title="Read the display">
    Once the game starts, the screen shows `3.` followed by the digits you have correctly entered so far (up to the last five are visible). A blinking underscore marks the position you need to fill next.
  </Step>
  <Step title="Press the next digit">
    Type the digit using your keyboard — only the keys `0` through `9` are recognised. The app compares your key press to the actual next digit of π derived from `PI_DIGITS` in `constants.ts`.
  </Step>
  <Step title="Watch the feedback">
    A correct key triggers a green border flash on the game panel and a short typing beep (generated by the Web Audio API). A wrong key triggers a red border flash and a lower-pitched error tone, and costs you a life.
  </Step>
  <Step title="Reach the goal or run out of lives">
    The game ends when you win (Normal mode only, at 50 digits), your time expires (Time mode), or you lose your last life.
  </Step>
</Steps>

## Game modes

<Tabs>
  <Tab title="Normal">
    **Normal mode** is the standard experience. You start with **3 lives** represented by red glowing circles in the top-right corner of the HUD. Your goal is to enter **50 consecutive correct digits**. Each wrong keystroke removes one life; when all three are gone the game ends. Reach digit 50 with at least one life remaining and you win, triggering a confetti burst and a two-tone success sound.
  </Tab>
  <Tab title="Time (Contra Reloj)">
    **Time mode** (Contra Reloj) puts you under a **30-second countdown** displayed in the center of the HUD. Crucially, **wrong key presses do not cost lives** in this mode — you can make as many errors as you like without penalty. Your only constraint is the clock. When the timer hits zero, the game ends and your score is the number of digits you entered correctly. The countdown turns red and pulses when fewer than 10 seconds remain.
  </Tab>
  <Tab title="Endless (Muerte Súbita)">
    **Endless mode** (Muerte Súbita — "Sudden Death") gives you **1 life** and **no digit limit**. A single wrong key press ends the game immediately. There is no winning condition; the goal is simply to push your personal best as far into π as possible. The HUD shows only one red circle instead of three, reminding you there is no margin for error.
  </Tab>
</Tabs>

## Scoring

Every correct digit awards **+10 points**, displayed live in the top-left of the HUD under the "Puntos" label. There is no bonus for speed or streak, so the final score is always `correct digits × 10`.

## Lives display

Lives are shown as a row of small circles in the top-right corner of the HUD. A filled red circle with a red glow represents an active life; a dim outlined circle represents a lost life. The circles pulse gently with a repeating Framer Motion animation while they are active.

| Filled circles | Meaning |
|---|---|
| 3 | Full health (Normal mode start) |
| 1 | One life remaining — one mistake ends the game |
| 0 | Game over |

In Endless mode only one circle is ever rendered, starting filled and becoming empty on the first error.

## Visual feedback

The game panel border and background change color instantly on every key press to give you unambiguous feedback:

| Key press | Panel border | Background tint | Sound |
|---|---|---|---|
| Correct | Green (`border-green-500`) | None | Short typing beep via Web Audio API |
| Wrong | Red (`border-red-500`) | Dark red (`bg-red-950/20`) | Two-tone descending sawtooth tone |

The green flash lasts **200 ms**; the red flash lasts **400 ms**. Both are driven by a `flash` state variable set on key press and cleared by a `setTimeout`.

## Game states

The component tracks four distinct states using the `GameStatus` type:

```typescript
type GameStatus = 'idle' | 'playing' | 'gameover' | 'won';
```

And three modes using the `GameMode` type:

```typescript
type GameMode = 'normal' | 'time' | 'endless';
```

| State | What you see |
|---|---|
| `idle` | The three mode selection cards |
| `playing` | The HUD, digit display, and hidden input |
| `gameover` | A red "Fin del Juego" screen with your digit count and Retry / Menu buttons |
| `won` | A glowing "¡Increíble!" screen with confetti, shown only in Normal mode |

## The digit source

The game pulls its reference string from `PI_DIGITS` in `src/lib/constants.ts` — a 1,000-character string beginning with `3141592653...`. The game strips the leading `3` and challenges you on the decimal portion starting at `1`, `4`, `1`, `5`, `9`, `2`, …

<Warning>
  The hidden input field that captures your key presses is always kept focused while the game is in the `playing` state. If you click outside the browser window and return, focus is automatically restored so you never miss a keystroke.
</Warning>

<Tip>
  Start with **Time mode** if you are new to memorising π. Because errors carry no life penalty, you can build familiarity with the digit sequence under pressure without an abrupt game-over screen interrupting your practice.
</Tip>
